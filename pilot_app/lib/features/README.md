# Features

Cada feature segue a estrutura:

- `data/` — API clients, DTOs, repositórios concretos
- `domain/` — entidades, interfaces de repositório, use cases
- `presentation/` — páginas, widgets, estado (Bloc/Cubit ou Provider)

Exemplos: `auth`, `route_planning`, `tracking`, `incidents`.
