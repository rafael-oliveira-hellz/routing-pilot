# APP-8001: Observabilidade — traceId e logging

## História

Como **suporte**, quero que erros exibam um código (traceId) para rastrear no backend; como **equipe**, queremos logs úteis em debug sem expor dados sensíveis.

## Critério de aceite

- traceId em todas as requisições (já em APP-1003); em telas de erro exibir "Código: {traceId}"; logs em produção sem tokens/senhas/posições.

## Subtasks

- [ ] Gerar traceId (UUID v4) no app para cada "sessão" de envio ou por requisição crítica
- [ ] Enviar em todas as requisições HTTP: header X-Trace-Id (já previsto em rede)
- [ ] Guardar traceId atual em memória (ou logging) para correlacionar com erros
- [ ] Em telas de erro, exibir "Código: {traceId}" para suporte rastrear no backend (execution_event, DLQ)
- [ ] Não logar tokens, senhas nem posições em produção
- [ ] Em debug: logs úteis (traceId, vehicleId, status da requisição, duração)
- [ ] (Opcional) Encoder JSON para logs estruturados, alinhado ao backend
