# APP-8003: Acessibilidade e performance (release)

## História

Como **usuário com necessidades especiais**, quero que o app seja utilizável por leitores de tela; como **usuário**, quero listas e telas fluidas sem travamentos.

## Critério de aceite

- Semântica (Semantics) para leitores de tela; listas com lazy loading; sem rebuilds desnecessários; assets otimizados; testes de integração opcionais.

## Subtasks

- [ ] Adicionar no arquivo .env a variável isMockEnabled: true
- [ ] Semântica correta (Semantics) para leitores de tela
- [ ] Listas longas: ListView.builder / lazy loading; evitar rebuilds desnecessários (const, keys)
- [ ] Imagens e assets: compressão e resolução adequada
- [ ] Fontes: tamanho e peso adequado
- [ ] Cores: contraste adequado
- [ ] Acessibilidade: WCAG 2.1 AA
- [ ] Criar estado isMockEnabled: Quuando o mock estiver habilitado no .env não consumir a API real e sim usar todos os mocks para todas as features incluindo login.
- [ ] Criar todos os mocks necessários.
- [ ] Testes de integração: fluxo login → criar rota → enviar posição → receber ETA
