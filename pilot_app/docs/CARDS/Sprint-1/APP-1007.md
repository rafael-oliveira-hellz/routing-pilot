# APP-1007: Roles (USER/ADMIN) e Admin — Lista de usuários

## História

Como **admin** (dono da aplicação), quero ver a lista de usuários e poder desativar/remover usuários; como **usuário comum**, não devo ver menu ou telas de administração.

## Critério de aceite

- Modelo de user inclui role; menu/telas admin só visíveis para role == ADMIN; tela lista de usuários (GET /users, DELETE) com confirmação; 403 tratado com "Sem permissão".

## Subtasks

- [x] **Roles no app:** dois papéis — USER e ADMIN; UserResponse.role (string) e getter isAdmin; JWT/backend envia role
- [x] **Admin-only (Flutter):** menu "Administração" (ícone usuários) só quando getCurrentUser()?.isAdmin == true; em 403 ApiClient rejeita com AuthException "Sem permissão"; tela /admin/users mostra "Sem permissão" se não for ADMIN
- [x] **Tela Admin — Lista de usuários:** GET /api/v1/users; listar (name, email, role, active); DELETE /api/v1/users/{id} com confirmação; não exibir botão remover para o próprio usuário; backend impede último admin/remover a si
- [x] **Demais features admin-only:** role vem do auth state (getCurrentUser()); rotas e itens de menu escondidos para USER; futuras telas admin (auditoria, DLQ, métricas) seguir mesmo padrão
