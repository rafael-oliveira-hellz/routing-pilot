# APP-1001: Estrutura do projeto e configuração

## História

Como **desenvolvedor**, quero uma estrutura de pastas clara e configuração por ambiente para que o app escale por feature e funcione em local/dev/staging/prod.

## Critério de aceite

- Estrutura `lib/core/`, `lib/features/`, `lib/shared/` existente.
- Navegação e DI configurados; app roda e carrega ambiente conforme config.

## Subtasks

- [x] Estrutura de pastas: `lib/core/`, `lib/features/`, `lib/shared/`
- [x] `core/`: `config/`, `di/`, `error/`, `network/`, `security/`, `theme/`, `utils/`
- [x] `features/`: um módulo por feature (auth, route_planning, tracking, incidents, etc.)
- [x] Por feature: `data/` (api, models, repositories), `domain/` (entities, repositories interfaces, use cases), `presentation/` (pages, widgets, bloc/cubit ou provider)
- [x] Navegação: go_router ou auto_route com rotas nomeadas e deep links
- [x] Injeção de dependência: get_it + injectable (ou provider) registrando APIs, repositórios, use cases
- [x] `flutter_dotenv` ou `--dart-define` para base URL, env (dev/staging/prod)
- [x] Configuração de timeouts, retry e tamanho máximo de payload (100 KB) alinhada ao backend
- [x] Constantes: intervalos de envio GPS (3–5 s movimento, 1 s curva, 10 s parado, 30 s heartbeat), batch max 10 posições ou 5 s
