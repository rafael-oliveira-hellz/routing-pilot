# APP-1003: Segurança e rede (storage, JWT, Dio, traceId)

## História

Como **usuário**, quero que tokens e dados sensíveis fiquem armazenados de forma segura e que todas as requisições usem autenticação e rastreio (traceId) para segurança e suporte.

## Critério de aceite

- Tokens em secure storage; Dio com interceptors (Bearer, X-Trace-Id); tratamento de 401/429/503; ErrorResponse tipado.

## Subtasks

- [x] Armazenamento seguro: `flutter_secure_storage` para tokens (access + refresh)
- [x] JWT: parse de claims (`vehicleId`/`sub`, exp), refresh antes de expirar (ex.: 1 h)
- [x] HTTPS obrigatório; certificate pinning (opcional, configurável por env)
- [x] Não logar tokens nem dados sensíveis; obfuscar em builds release
- [x] Cliente HTTP: Dio com interceptors para: Auth (Bearer), X-Trace-Id, Content-Type, timeout
- [x] Geração e propagação de `traceId` (UUID v4) em todas as requisições para o backend
- [x] Tratamento de 401 (refresh ou logout), 429 (Retry-After), 503 (backoff)
- [x] Tipagem de erros (ErrorResponse: timestamp, status, errorCode, message, path, traceId)
